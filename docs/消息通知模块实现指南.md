## 1. 整体流程概览

### 1.1 通知类型

```
APPLICATION_UPDATE  — 投递状态变更（已查看/筛选通过/不合适/录用）
INTERVIEW_INVITE    — 面试邀请（招聘方安排面试后自动发）
MESSAGE             — 收到新私信提醒
SYSTEM              — 系统公告/管理员推送
```

### 1.2 通知流程

```
业务动作发生(投递状态变更/面试安排/新私信)
  → Service 调用 notificationService.create(userId, type, title, content, relatedId)
  → 写入 notification 表
  → 前端轮询/查询 GET /api/notifications
  → 用户点击标记已读 PUT /api/notifications/{id}/read
```

### 1.3 私信流程

```
发送方选择接收方 → 输入内容 → POST /api/messages { receiverId, content, applicationId? }
  → 校验发送方和接收方存在
  → 插入 message 表
  → 自动创建一条 MESSAGE 类型通知给接收方
  → 返回消息详情

接收方查看:
  GET /api/messages/conversations → 会话列表（按发送方分组，取最新一条）
  GET /api/messages/{userId} → 与某用户的完整对话
```

### 1.4 消息投递联动

投递模块中，状态变更时自动创建通知：
```
PUT /api/applications/{id}/status
  → 更新 application.status
  → notificationService.create(seekerId, "APPLICATION_UPDATE", "投递状态更新", "您的投递已被HR查看...", applicationId)
```

面试安排时自动创建通知：
```
POST /api/applications/{id}/interview
  → 插入 interview 表
  → notificationService.create(seekerId, "INTERVIEW_INVITE", "面试邀请", "您收到一份面试邀请...", applicationId)
```

```java
package com.li.ai_job_market.model.entity;

@Data

    @TableField(exist = false) private static final long serialVersionUID = 1L;
}
```

## 5. 第 4 步：Service (2 个)

### 5.1 NotificationService

```java
public interface NotificationService extends IService<Notification> {
    /** 创建通知 */
    void create(Long userId, String type, String title, String content, Long relatedId);
    /** 我的通知列表（分页） */
    Page<NotificationVO> listMyNotifications(Long userId, int current, int size);
    /** 未读数量 */
    long unreadCount(Long userId);
    /** 标记已读 */
    boolean markRead(Long id, Long userId);
    /** 全部标记已读 */
    boolean markAllRead(Long userId);
    /** 删除通知 */
    boolean delete(Long id, Long userId);
}
```

### 5.2 MessageService

```java
public interface MessageService extends IService<Message> {
    /** 发送私信 */
    MessageVO send(Long senderId, Long receiverId, Long applicationId, String content);
    /** 与某人的对话列表（分页） */
    Page<MessageVO> getConversation(Long userId, Long peerId, int current, int size);
    /** 会话列表（按对方分组，每组取最新一条） */
    List<MessageVO> listConversations(Long userId);
    /** 未读消息数 */
    long unreadCount(Long userId);
    /** 标记已读 */
    boolean markRead(Long id, Long userId);
}
```

### 5.3 实现要点

**NotificationServiceImpl**：
- `create()` — 直接 insert，type 不区分大小写但在前端做中文映射
- `listMyNotifications()` — 按 userId 查询，`orderByDesc(createdAt)`，类型标签映射
- `markRead()` — `update().set("is_read", 1).eq("id", id).eq("user_id", userId)`
- `markAllRead()` — `update().set("is_read", 1).eq("user_id", userId).eq("is_read", 0)`

**MessageServiceImpl**：
- `send()` — insert 后自动调用 `notificationService.create(receiverId, "MESSAGE", ...)`
- `getConversation()` — `(sender_id = userId AND receiver_id = peerId) OR (sender_id = peerId AND receiver_id = userId)` 按时间升序
- `listConversations()` — 用子查询或 GROUP BY 查出每个 peer 的最新一条消息
- `markRead()` — 标记对方发来的消息为已读

## 6. 第 5 步：Controller (1 个)

### NotificationController.java

```java
@Slf4j
@RestController
@RequestMapping("/api")
public class NotificationController {

    @Resource private NotificationService notificationService;
    @Resource private MessageService messageService;
    @Resource private UserService userService;

    // ========== 通知 ==========
    @GetMapping("/notifications")          // 我的通知列表（分页）
    @GetMapping("/notifications/unread-count")  // 未读通知数
    @PutMapping("/notifications/{id}/read")    // 标记单条已读
    @PutMapping("/notifications/read-all")      // 全部已读
    @DeleteMapping("/notifications/{id}")        // 删除通知

    // ========== 私信 ==========
    @PostMapping("/messages")                      // 发送私信
    @GetMapping("/messages/conversations")         // 会话列表
    @GetMapping("/messages/{peerId}")              // 与某人的对话
    @GetMapping("/messages/unread-count")          // 未读消息数
    @PutMapping("/messages/{id}/read")             // 标记已读
}
```

共 **10 个端点**，统一通过 `getLoginUser(request)` 获取当前用户。

```bash
mvnw clean compile -DskipTests
```

### 8.2 API 测试（登录后）

```bash
# 通知列表
curl -H "Authorization: Bearer $TOKEN" http://localhost:8123/api/notifications
```
