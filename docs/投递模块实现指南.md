## 1. 整体流程概览

### 1.1 投递生命周期

```
求职者投递 → APPLIED ──HR查看──→ VIEWED ──筛选通过──→ SCREENING
                                        ↓ 不合适
                                     REJECTED
                    ↓
              安排面试 → INTERVIEW ──通过──→ OFFER ──入职──→ HIRED
                              ↓ 不通过
                           REJECTED

任何阶段求职者可主动撤回: WITHDRAWN
```

### 1.2 投递流程

```
POST /api/applications { jobId, resumeId, coverLetter? } → Service
  → 校验角色(SEEKER)
  → 查 job 确认 PUBLISHED
  → 查 resume 确认归属
  → 自动计算 AI 匹配度(技能重合)
  → 插入 application(status=APPLIED)
  → 更新 job.apply_count +1
  → 写入 application_log
  → 返回投递记录
```

### 1.3 状态变更流程

```
PUT /api/applications/{id}/status { status, remark? } → Service
  → 校验 RECRUITER (招聘方本人或管理员)
  → 查旧状态
  → 更新 application.status
  → 写入 application_log(from_status → to_status)
  → 返回成功
```

### 1.4 面试安排流程

```
POST /api/applications/{id}/interview { type, time, duration, location... } → Service
  → 校验 RECRUITER
  → application.status 改为 INTERVIEW
  → 插入 interview 表
  → 写入 application_log
  → 返回面试信息
```

### 1.5 收藏流程

```
POST /api/favorites { targetType: "JOB"|"RESUME", targetId } → Service
  → 校验已登录
  → 检查唯一约束(user_id + target_type + target_id)
  → 已存在则忽略，不存在则插入
  → 返回收藏记录 ID

DELETE /api/favorites/{id} → 删除(校验归属)
GET    /api/favorites → 我的收藏列表
```

### 1.6 关键设计决策

| 决策点 | 方案 | 原因 |
|--------|------|------|
| 匹配度计算时机 | 投递时自动计算（不做持久化向量） | 简化实现，后续可升级 |
| 状态日志 | 每次变更自动写入 application_log | 完整的可追溯审计 |
| 收藏唯一性 | DB 层联合唯一索引 | 防止重复，数据库兜底 |
| 面试安排 | 合并状态变更+面试创建 | 减少 API 调用，事务保证 |
| 权限控制 | SEEKER 投递，RECRUITER 操作状态，双方可查看 | 职责清晰 |

public long apply(Long seekerId, Long jobId, Long resumeId, String coverLetter) {
    // 1. 校验职位存在且 PUBLISHED
    Job job = jobMapper.selectById(jobId);
    ThrowUtils.throwIf(job == null || !"PUBLISHED".equals(job.getStatus()),
            ErrorCode.NOT_FOUND_ERROR, "职位不可投递");

    // 2. 校验简历归属
    Resume resume = resumeMapper.selectById(resumeId);
    ThrowUtils.throwIf(resume == null || !resume.getUserId().equals(seekerId),
            ErrorCode.NO_AUTH_ERROR, "无权使用此简历");

    // 3. 计算 AI 匹配度（技能重合）
    // 4. 创建投递记录
    // 5. job.apply_count +1
    // 6. 写入 application_log
}

## 6. 第 5 步：Controller (1 个)

### 6.1 `ApplicationController.java`

```java
@RestController
@RequestMapping("/applications")
public class ApplicationController {

    @Resource private ApplicationService applicationService;
    @Resource private UserService userService;

    // 投递
    @PostMapping
    public BaseResponse<Long> apply(@RequestBody ApplicationRequest req, HttpServletRequest request) {
        UserVO user = userService.getLoginUser(request);
        return ResultUtils.success(applicationService.apply(user.getId(), req.getJobId(), req.getResumeId(), req.getCoverLetter()));
    }

    // 列表
    @GetMapping("/my")
    public BaseResponse<Page<ApplicationVO>> my(@RequestParam(defaultValue="1") int current, @RequestParam(defaultValue="10") int size, HttpServletRequest request) {
        UserVO user = userService.getLoginUser(request);
        return ResultUtils.success(applicationService.listMyApplications(user.getId(), current, size));
    }

    @GetMapping("/received")
    public BaseResponse<Page<ApplicationVO>> received(@RequestParam(defaultValue="1") int current, @RequestParam(defaultValue="10") int size, @RequestParam(required=false) String status, HttpServletRequest request) {
        UserVO user = userService.getLoginUser(request);
        return ResultUtils.success(applicationService.listReceivedApplications(user.getId(), current, size, status));
    }

    // 详情
    @GetMapping("/{id}")
    public BaseResponse<ApplicationVO> detail(@PathVariable Long id, HttpServletRequest request) {
        UserVO user = userService.getLoginUser(request);
        return ResultUtils.success(applicationService.getDetail(id, user.getId()));
    }

    // 状态变更
    @PutMapping("/{id}/status")
    public BaseResponse<Boolean> updateStatus(@PathVariable Long id, @RequestBody ApplicationStatusRequest req, HttpServletRequest request) {
        UserVO user = userService.getLoginUser(request);
        return ResultUtils.success(applicationService.updateStatus(id, user.getId(), req.getStatus(), req.getRemark()));
    }

    // 面试
    @PostMapping("/{id}/interview")
    public BaseResponse<Interview> scheduleInterview(@PathVariable Long id, @RequestBody InterviewRequest req, HttpServletRequest request) {
        UserVO user = userService.getLoginUser(request);
        return ResultUtils.success(applicationService.scheduleInterview(id, user.getId(), req));
    }

    @PutMapping("/{id}/interview/{iid}/feedback")
    public BaseResponse<Boolean> addFeedback(@PathVariable Long id, @PathVariable Long iid, @RequestParam String feedback, HttpServletRequest request) {
        UserVO user = userService.getLoginUser(request);
        return ResultUtils.success(applicationService.updateInterviewFeedback(iid, user.getId(), feedback));
    }

    // 收藏
    @PostMapping("/favorites")
    public BaseResponse<Long> addFavorite(@RequestBody FavoriteRequest req, HttpServletRequest request) {
        UserVO user = userService.getLoginUser(request);
        return ResultUtils.success(applicationService.addFavorite(user.getId(), req.getTargetType(), req.getTargetId()));
    }

    @DeleteMapping("/favorites/{id}")
    public BaseResponse<Boolean> removeFavorite(@PathVariable Long id, HttpServletRequest request) {
        UserVO user = userService.getLoginUser(request);
        return ResultUtils.success(applicationService.removeFavorite(id, user.getId()));
    }

    @GetMapping("/favorites")
    public BaseResponse<List<FavoriteVO>> listFavorites(HttpServletRequest request) {
        UserVO user = userService.getLoginUser(request);
        return ResultUtils.success(applicationService.listFavorites(user.getId()));
    }
}
```
